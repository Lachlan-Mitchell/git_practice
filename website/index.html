Setting up a branch

Navigate using cd (followed by the directory(folder) you want to move to). You can also use cd . to move down through the folder and cd .. to move up.

After locating the correct directory you can use git brach followed by the branch name you would like to create your own branch. The benefits of having a seperate branch allows you to test and create new code without impacting the main branch. The main branch should be (and is most commonly used) for the live product code


The process of editing your branch

Before you run straight into the thick of it, ensure that you have used git merge main. This ensures that your branch is uptodate with the main branch. Now you may edit the code to your hearts content without worry of damaging anything and knowing that you are editing code that is up to date. Once you are ready to commit your changesyou will:



  1. git add . this will addd all changes to the staging phase, ready to be commited.
  2. git commit -m "messgae goes here"will commit your changes to the branch(like saving them)
  3. git pushWill add any commited changes to the branch
  4. Create a pull request on GitHub this is then reviewed and then can be added to the main branch if approved
